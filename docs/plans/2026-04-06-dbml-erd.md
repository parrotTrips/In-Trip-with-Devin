# DBML ERD Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create a lean DBML artifact for the current trip database model so the ERD can be visualized and validated in DBML tooling.

**Architecture:** The implementation is documentation-first. It converts the approved logical model into a single DBML file under `docs/db/`, preserving tables, foreign keys, and the uniqueness rules that affect cardinality and ERD readability. It intentionally avoids detailed DDL concerns such as checks and verbose notes.

**Tech Stack:** DBML, Markdown documentation
